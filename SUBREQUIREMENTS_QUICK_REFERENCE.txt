## Sub-Requirements Feature - Quick Reference Card

### 🎯 Quick Access
- **Dashboard URL:** `http://localhost/PDF_Reader/uat_test_cases_enhanced.php`
- **Feature:** Separate test cases for each sub-requirement (FR-20.01, FR-20.02, etc)

### 🧪 Test Execution

**Before clicking Execute:**
- Each sub-requirement has 4 separate test cards
- Clicking "Execute →" links to exact sub-requirement

**Example URLs:**
- `uat_test_execution.php?req=FR-20.01&tc=FR-20.01_TC_001`
- `uat_test_execution.php?req=FR-20.02&tc=FR-20.02_TC_002`
- `uat_test_execution.php?req=FR-20.03&tc=FR-20.03_TC_003`

### 💡 Usage Tips

1. **Expand parent requirement** to see all sub-requirements
2. **Search by sub-requirement code** (e.g., "FR-20.01") to find specific tests
3. **Filter by test type** to focus on one test category
4. **Execute sub-requirement tests in order** (FR-20.01 → FR-20.02 → FR-20.03)
5. **Track coverage per sub-requirement** in results dashboard

### 📞 Troubleshooting

**No sub-requirements showing?**
- Check: PDF has properly formatted sub-items (FR-20.01: Description format)
- Verify: SRSParser extracted content array

**Wrong test IDs showing?**
- Check: ComprehensiveTestCaseGenerator updated
- Verify: Test case generation includes sub-requirement code

**Filtering not working?**
- Refresh browser page
- Check browser console for JavaScript errors
- Try different filter combination
