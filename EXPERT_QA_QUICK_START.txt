🎯 TEST CASE TYPES EXPLAINED
═════════════════════════════════════════════════════════════════════════════════

✅ POSITIVE Test (TC_001)
   What: Happy path execution
   Tests: "What should happen when everything is correct?"
   Example: Author successfully uploads camera-ready file
   Status: 00% system works as designed

❌ NEGATIVE Test (TC_002)
   What: Error handling
   Tests: "How does system handle invalid input?"
   Example: Author uploads wrong file format
   Status: System rejects gracefully with error message

🔄 EDGE CASE Test (TC_003)
   What: Boundary conditions
   Tests: "What happens at system limits?"
   Example: Upload exactly at deadline, concurrent users
   Status: System handles limits, no overflow/timeout

⚡ PERFORMANCE Test (TC_004) - NFRs only
   What: Performance validation
   Tests: "Does system meet speed/load requirements?"
   Example: 4-track merge completes in ≤6 minutes
   Status: Response time, throughput, resource use

🔀 ALTERNATE FLOW Test (TC_004) - FRs only
   What: Workflow variations
   Tests: "Other paths through the requirement?"
   Example: Email sent to all co-authors on upload
   Status: Different actors, status updates, notifications

🏗️ TEST CASE STRUCTURE
═════════════════════════════════════════════════════════════════════════════════

Each test case includes:

Identification:
  • Test ID: FR-20.01_TC_001 (maps to requirement + test number)
  • Requirement ID: FR-20.01 (SRS mapping)
  • Test Name: [POSITIVE] Author Registers and Pays
  • Test Type: Positive, Negative, Edge Case, Performance, or Alternate

Context:
  • Description: What this test verifies
  • User Persona: Author, Manager, Admin, System, etc.
  • Priority: P0 (Critical), P1 (High), P2 (Medium)

Execution:
  • Pre-conditions: "Author logged in, payment method configured"
  • Test Steps: 1. Author navigates.. 2. System.. 3. Author..
  • Expected Result: "Payment processed, confirmation sent"

Validation:
  • Acceptance Criteria: Checkable success criteria
  • Test Data: Sample input values (email, file, amount)
  • Automation Ready: Yes/No flag
