📋 TEST CASE GENERATION LOGIC
─────────────────────────────────────────────────────────────────────────────────

For each Functional Requirement (FR):
   1. Positive Test (T1)      → Basic functionality
   2. Edge Case Test (T2)     → Boundary conditions  
   3. Negative Test (T3)      → Error handling

For each Non-Functional Requirement (NFR):
   1. Positive Test (T1)      → Basic functionality
   2. Edge Case Test (T2)     → Boundary conditions
   3. Negative Test (T3)      → Error handling
   4. Performance Test (T4)   → If performance-related

Total: 3-4 test cases automatically generated per requirement!
